🚨 Critical

CRITICAL ไม่มีระบบ Authentication / Authorization

สาเหตุ: API ทุก endpoint เปิดให้เข้าถึงได้โดยไม่ต้องล็อกอิน ใครก็ตามที่รู้ URL สามารถอ่าน สร้าง แก้ไข หรือลบ Case ได้ทั้งหมด

// ปัจจุบัน — ไม่มีการตรวจสอบตัวตน app.delete('/api/cases/:id', (req, res) => { // ทุกคนลบได้เลย! });
✅ วิธีแก้ไข: เพิ่ม JWT Authentication middleware และจำกัดสิทธิ์ตาม role (Admin/Staff)
const jwt = require('jsonwebtoken'); const auth = (req, res, next) => { const token = req.headers.authorization?.split(' ')[1]; if (!token) return res.status(401).json({ error: 'Unauthorized' }); try { req.user = jwt.verify(token, process.env.JWT_SECRET); next(); } catch { res.status(403).json({ error: 'Invalid token' }); } }; app.delete('/api/cases/:id', auth, (req, res) => { ... });

⚠️ High

HIGH ไม่มี Input Sanitization (XSS Risk)

สาเหตุ: Frontend แสดงข้อมูลจาก API โดยไม่ escape HTML ถ้าผู้ใช้บันทึกข้อมูลที่มี script tag ก็อาจถูก execute ได้

// ปัจจุบัน — อันตราย td.innerHTML = caseData.subject; // subject อาจมี <script>
✅ วิธีแก้ไข: ใช้ textContent แทน innerHTML หรือ escape HTML ก่อนแสดงผล
function escapeHtml(str) { return str.replace(/&/g,'&').replace(//g,'>').replace(/"/g,'"'); } td.textContent = caseData.subject; // ปลอดภัย
HIGH ไม่มี Rate Limiting

สาเหตุ: API ไม่จำกัดจำนวน request ต่อนาที ทำให้ถูก brute force หรือ DoS attack ได้ง่าย

✅ วิธีแก้ไข: ติดตั้ง express-rate-limit
const rateLimit = require('express-rate-limit'); app.use('/api/', rateLimit({ windowMs: 15*60*1000, max: 100 }));

🔶 Medium

MEDIUM ข้อมูล Sensitive ไม่ถูก Mask

สาเหตุ: เบอร์โทรและข้อมูลติดต่อลูกค้าถูกส่งกลับในทุก API response โดยไม่มีการ mask เช่น 02-111-2222 → 02-111-XXXX

✅ วิธีแก้ไข: Mask ข้อมูลในฝั่ง API หรือกำหนด field access ตาม role
MEDIUM ไม่มี HTTPS (ใช้ HTTP)

สาเหตุ: Server รันบน HTTP ข้อมูลถูกส่งแบบ plaintext อาจถูก intercept ได้ถ้า deploy บน network

✅ วิธีแก้ไข: ใช้ nginx reverse proxy กับ SSL certificate หรือ deploy บน platform ที่มี HTTPS เช่น Railway, Render
MEDIUM ไม่มี CORS Restriction

สาเหตุ: ใช้ cors() แบบ default อนุญาตทุก origin ทำให้ website ภายนอกสามารถเรียก API ได้

app.use(cors()); // เปิดทุก origin — ไม่ปลอดภัย
✅ วิธีแก้ไข: จำกัด origin ที่อนุญาต
app.use(cors({ origin: 'http://localhost:3000' }));

🟢 Low

LOW ไม่มี Security Headers (Helmet)

สาเหตุ: Response ไม่มี HTTP security headers เช่น X-Content-Type-Options, X-Frame-Options ทำให้เสี่ยงต่อ clickjacking

✅ วิธีแก้ไข:
const helmet = require('helmet'); app.use(helmet());
